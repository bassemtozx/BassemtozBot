# بوت إدارة المحاضر — Discord Case System

بوت ديسكورد لإدارة الشكاوى والمحاضر داخل سيرفر واحد. المستخدمون يرفعون محاضر، المشرفون يديرونها، والقضاة/الأدمن يصدرون الحكم.

## المتطلبات

- Python 3.11+
- discord.py 2.x
- sqlite3 (مدمج مع بايثون)

## التثبيت والتشغيل

1. استنساخ أو فك المشروع ثم فتح المجلد.
2. إنشاء بيئة افتراضية (موصى به):
   ```bash
   python -m venv venv
   venv\Scripts\activate
   ```
3. تثبيت الحزم:
   ```bash
   pip install -r requirements.txt
   ```
4. نسخ `.env.example` إلى `.env` وملء القيم:
   ```bash
   copy .env.example .env
   ```
5. في `.env` ضبط:
   - `DISCORD_TOKEN`: توكن البوت من [Discord Developer Portal](https://discord.com/developers/applications).
   - `GUILD_ID`: (اختياري) معرف السيرفر لتسريع مزامنة الأوامر على سيرفر واحد.
   - `STAFF_ROLE_ID`: معرف رول المشرفين (يديرون القضايا).
   - `JUDGE_ROLE_ID`: معرف رول القضاة (يصدرون الحكم).
   - `ADMIN_ROLE_ID`: معرف رول الأدمن (يصدرون الحكم ويفتحون قفل الحكم).
   - `OPEN_CASES_CATEGORY_ID`: معرف فئة القنوات للقضايا المفتوحة (تُنشأ فيها قنوات القضايا).
   - `CLOSED_CASES_CATEGORY_ID`: معرف فئة القنوات للمغلقة/المؤرشفة (نقل القناة عند الإغلاق أو الأرشفة).
   - `VERDICTS_CHANNEL_ID`: قناة إعلان الأحكام (اختياري).
   - `LOG_CHANNEL_ID`: قناة السجلات (اختياري).
   - `DB_PATH`: مسار ملف sqlite (افتراضي: `cases.db`).
6. تشغيل البوت:
   ```bash
   python bot.py
   ```

## إعداد السيرفر

1. إنشاء الرولات: مشرفين، قضاة، أدمن (أو استخدام رولات موجودة).
2. إنشاء فئتين للقنوات: واحدة للقضايا المفتوحة وأخرى للمغلقة.
3. إنشاء قناة لرفع المحاضر (مثلاً `#رفع-محضر`).
4. تشغيل الأمر:
   - `/setup_cases channel:#رفع-محضر`
   سيُرسل البوت في القناة المختارة رسالة مع زر **رفع محضر**.

| `/case_open_manual` | فتح قضية يدوياً | مشرف |
| `/case_view` | عرض تفاصيل قضية | صاحب القضية أو المشرف |
| `/case_list` | قائمة القضايا (مع تصفية اختيارية) | الكل |

| `/case_assign_judge` | تعيين قاضٍ | مشرف |
| `/case_add_defendant` | إضافة المتهم (عضو) للقضية والقناة | مشرف |
| `/case_status` | تغيير حالة القضية | مشرف |
| `/case_note` | إضافة ملاحظة إدارية | مشرف |
| `/case_verdict` | إصدار الحكم النهائي | قاضي/أدمن |
| `/case_close` | إغلاق القضية | مشرف |
| `/case_archive` | أرشفة القضية (وتصدير النقل إن أمكن) | مشرف |
| `/case_reopen` | إعادة فتح القضية | مشرف |
| `/case_logs` | عرض سجل القضية | صاحب القضية أو مشرف |
| `/case_appeal_review` | قبول أو رفض الاستئناف | قاضي/أدمن |
| `/case_override_unlock_verdict` | فتح قفل الحكم | أدمن فقط |

## سير العمل

1. المستخدم يضغط **رفع محضر** ويملأ النموذج (المشتكى عليه، نوع القضية، الوصف، الأدلة، الشهود).
2. يُنشأ رقم قضية (مثل CASE-1001) وقناة خاصة بها؛ الوصول: مقدم الشكوى، المشرفون، القضاة، الأدمن.
3. المشرف يستلم القضية، يعين قاضياً، يضيف أدلة، يغير الحالة، يكتب ملاحظات.
4. القاضي/الأدمن يصدر الحكم (نوع الحكم، السبب، مدة العقوبة، هل الاستئناف متاح).
5. بعد الحكم يُقفل الحكم ويُعلن في قناة الأحكام (إن وُجدت).
6. إن كان الاستئناف متاحاً، مقدم الشكوى يقدم استئنافاً؛ القاضي/الأدمن يقبل أو يرفض.

## التشغيل 24/7

- يمكن تشغيل البوت على VPS أو خدمة استضافة (مثل systemd، Docker، أو منصة Bot hosting).
- بعد إعادة تشغيل البوت، زر **رفع محضر** في قناة الإعداد يعمل إذا تم تسجيل الـ view عند التحميل؛ أزرار القضايا داخل قنوات القضايا تحتاج أوامر الـ slash للتعامل مع القضية إن لم تكن الـ views محمّلة لتلك الرسائل.

## هيكل المشروع

- `bot.py`: نقطة الدخول ومزامنة الأوامر.
- `config.py`: تحميل الإعدادات من `.env`.
- `database/`: إنشاء الجداول واستعلامات sqlite (cases, case_logs, case_notes, case_appeals).
- `cogs/cases.py`: أوامر الـ slash ومعالجة رفع المحاضر والحكم والاستئناف.
- `views/`: أزرار الإعداد وأزرار إجراءات القضية.
- `modals/`: نماذج رفع المحضر، الحكم، الاستئناف، الملاحظة، الأدلة.
- `utils/`: صلاحيات، بناء الـ embeds، قناة السجل، تصدير النقل.
